Шаг 1. Введение

В одной из прошлых демонстраций мы работали с Git прямо в редакторе кода VS Code, то есть без использования команд. Тогда мы выяснили, что такой вариант в каких-то моментах хорошо подходит для работы над проектом, а в каких-то лишь усложняет процесс.

В этой демонстрации мы рассмотрим приложение от GitHub — GitHub Desktop. Через него можно создавать локальные и одновременно удалённые репозитории, клонировать удалённые репозитории, управлять ветками, просматривать историю репозитория и так далее. По функциональности это приложение мало чем отличается от встроенного в редактор кода VS Code. А тем, кто привык работать с продуктами компании JetBrains, например, с тем же WebStorm, возможности GitHub Desktop покажутся очень ограниченными.

Часто GitHub Desktop рекомендуют новичкам, потому что в нём можно запомнить кодовую фразу, использующуюся в SSH. Однако если вы настроите SSH-агент, то особой разницы не заметите.

Возможно, вам понравится интерфейс программы, и вы поэтому решите работать с ней. Но хотим предостеречь: ни одна программа не раскрывает полный потенциал Git, то есть вы всегда будете ограничены в возможностях. Да и к тому же на работе чаще требуют умение работать с Git через терминал, а не через редакторы кода или сторонние приложения.
